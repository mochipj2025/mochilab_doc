# Blog Editor Plan

M.O.C.H.I. LABOのブログサイトと執筆エディター構想。

単なる記事投稿システムではなく、世界観、キャラクター、Safe Base思想を育てるための執筆ツールとして設計する。

## Purpose

- エッセイ、コラム、研究ログを保存する
- ケツコラム部署の記事を育てる
- もちスラ、ケツまる、まじめちゃんの掛け合いを作りやすくする
- 医療表現や過度な広告表現を避ける
- 静かな日本語ミニマルUIで読みやすく公開する

## Blog Site

ブログサイトは、読む場所として静かに作る。

必要な要素:

- 記事一覧
- エッセイ一覧
- ケツコラム
- 研究ログ
- キャラクター登場回
- タグ
- モバイルで読みやすい本文レイアウト

優先する空気感:

- 静か
- 余白がある
- 読み疲れしない
- 広告感が少ない
- キャラクターが主張しすぎない

## Editor

エディターは、書く場所としてやさしく作る。

必要な要素:

- タイトル入力
- 本文入力
- カテゴリ選択
- タグ選択
- 登場キャラクター選択
- トーン選択
- Markdownプレビュー
- 投稿テンプレート保存
- NG表現チェック

## Character Dialogue Support

掛け合いの基本ユニット:

- もちスラ
- ケツまる
- まじめちゃん

エディターでできるとよいこと:

- 3人掛け合いテンプレートを挿入する
- もちスラの構造化コメントを挿入する
- ケツまるの軽いツッコミを挿入する
- まじめちゃんの生活者目線の問いを挿入する
- 他の研究員をゲストとして追加する

## Essay Series Ideas

### SNSこわい

落語「まんじゅうこわい」の構造を使ったエッセイ。

扱うテーマ:

- 面白いエッセイが読めるのが怖い
- 儲かりそうな話題が見えて怖い
- たまにセンシティブも見たくなってしまって怖い
- 本当に怖いのは通知が来ないこと

### Future Ideas

- AIこわい
- バズこわい
- コメント欄こわい
- おすすめ欄こわい
- 収益化こわい

## Safety and Tone Checks

エディターには、以下の観点を入れる。

- 医療表現になっていないか
- 効果保証に見えないか
- 不安を煽っていないか
- コンプレックスを刺激していないか
- 広告っぽくなりすぎていないか
- モバイルで読みやすい長さか
- M.O.C.H.I. LABOの静かな空気感を保っているか

## Future Directory Idea

実装する場合の候補構成:

```txt
blog-editor/
├ README.md
├ stable/
├ dev/
├ experimental/
└ assets/
```

最初は `dev/` で小さく試し、安定したら `stable/` に移す。
