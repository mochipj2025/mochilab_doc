# UI Rules

## 基本

- モバイルファーストで設計する
- 1画面1目的を意識する
- 日本語は短く、自然な言い回しにする
- 重要な操作は画面下部でも扱いやすくする

## レイアウト

- 最小幅320pxで崩れないことを基準にする
- 余白は8px単位を基本にする
- カードやパネルは必要な場合だけ使う
- 主要アクションは視線と親指の届きやすさを考える

## タイポグラフィ

- 本文は読みやすいサイズを優先する
- 見出しは大きくしすぎない
- 長い説明文より、短いラベルと補助文を使う

## インタラクション

- タップ領域は十分に確保する
- ローディング、成功、失敗、空状態を用意する
- 危険な操作には確認を入れる
- ユーザーを責めるエラー文にしない

## コンポーネント

- ボタン、入力欄、カード、ナビゲーションは再利用できる形にする
- 色だけに依存せず、文言やアイコンでも状態を伝える
- stableに入れる前に複数画面で使えるか確認する
